Scheme Standards

The Scheme programming language (Wikipedia) was introduced in the 1975 paper, Scheme: An Interpreter for Extended Lambda Calculus. Since then it has been gradually improved and extended through many rounds of standardization.

The standardization process

The authoritative standards are the Scheme reports. Their names follow the convention Revisedn Report on the Algorithmic Language Scheme, abbreviated RnRS.

R3RS was the first standard that explicitly bore this name. The names of the early standards have been retroactively abbreviated using the same pattern: R2RS, R1RS, and R0RS.

The process is overseen by the Scheme Language Steering Committee.

Modern standards

Most active Scheme implementations now support one or both of these standards. Writing code that works on both R6RS and R7RS is possible with some care.

Revised7 Report on the Algorithmic Language Scheme (2013)

Abbreviated R7RS

R7RS brings back the simplicity of R5RS in the core language. Complexity has been moved into an optional large edition that is still a work in progress.

Revised6 Report on the Algorithmic Language Scheme (2007)

Abbreviated R6RS

R6RS abandoned the simplicity of R5RS and shipped a more full-featured language. Error situations are much more closely specified; a formal hierarchy of condition (exception) types is given.

Features procedural macros via syntax-case; hash tables; custom ports, transcoding and endianness; and a much larger standard library.

This is the first standard with a library system, enabling programmers to share libraries that work out-of-the-box on multiple Scheme implementations.

Classic standards

These standards present the core Scheme syntax and semantics in essentially the same form as it now is.

Revised5 Report on the Algorithmic Language Scheme (1998)

Abbreviated R5RS, this standard comes closest to a Platonic Form of Scheme before later standards tackle the complexities of programming in the large with features such as libraries and exceptions.

Weighing in at 50 pages, the R5RS document is an object of reverence among programming language enthusiasts. It is a touchstone of the Scheme community and is considered an exemplar of a concise and expressive language specification in other communities as well.

Early standards

The first three Scheme reports were research memos from the MIT Artificial Intelligence Laboratory (now CSAIL). They are written in an informal tone and are not as comprehensive or rigorous as the later reports.

The core language evolved throughout these reports. They are subtly incompatible with each other and with the current form of Scheme. No implementation since about 1990 is based on them.

Other standards

These documents are ratified by established standardization organizations. They exist outside the RnRS series, but nevertheless closely follow the RnRS designs of their time. Despite the fact that these standards come from reputable organizations, they are now rarely used, and mainly serve as historical curiosities. Virtually all Scheme programmers and implementers prefer the RnRS series.

IEEE Standard for the Scheme Programming Language (1990)

Abbreviated IEEE Scheme

IEEE 1178-1990

IEEE-ratified standard

IEEE Scheme is based on R4RS but incorporates a few R5RS features. For example, (eq? '() #f) is guaranteed to be false. The standard has no macros.
